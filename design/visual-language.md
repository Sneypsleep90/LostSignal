# Lost Signal — Visual Language

## Суть проекта

**Lost Signal** — это премиальный трекер цифровой тишины.

Это не приложение привычек.
Не productivity tool.
Не челлендж.
Не мотивационный трекер.

Lost Signal — это тихий интерфейс-путь, где пользователь уходит дальше от цифрового шума: коротких видео, ленты, новостей, соцсетей и постоянного информационного фона.

Главная идея:

> Уйти от шума.

Интерфейс должен ощущаться не как контроль, а как дистанция.
Не как “ты должен”, а как “ты уже дальше”.

---

# Главное ощущение

Lost Signal должен ощущаться как ночной северный лес за стеклом.

Ключевые состояния:

* тишина;
* туман;
* дистанция;
* слабый сигнал;
* холодный воздух;
* медленное движение вперёд;
* исчезающий цифровой шум;
* одиночный путь без лишних раздражителей.

Пользователь не “закрывает привычку”.
Он уходит глубже в место, где шума становится меньше.

---

# Визуальная метафора

Главная метафора проекта:

> Чем дольше пользователь вдали от шума, тем глубже он уходит в лес.

Каждый день — это не стрик.
Это расстояние.

Каждая локация — не уровень.
Это точка пути.

Каждый час, не потраченный на шум, — это возвращённое время.

---

# Текущий визуальный маршрут

Базовая тема проекта:

```txt
Шумный лес / Noisy Forest
```

Это основной визуальный мир Lost Signal на этапе MVP.

Образ:

* тёмный лес;
* силуэты елей;
* туман;
* почти исчезающий свет;
* стеклянные карточки;
* muted typography;
* зелёный сигнал как единственный акцент.

Тема должна быть тёмной, спокойной и кинематографичной.
Не хоррор.
Не fantasy.
Не game UI.
Не cyberpunk.

Это премиальный quiet interface.

# Цветовая система

Основная палитра:

```css
:root {
  --bg: #0d0f0e;
  --surface: rgba(255, 255, 255, 0.04);
  --border: rgba(255, 255, 255, 0.07);

  --text-primary: #e8e9e5;
  --text-muted: #5a6360;
  --text-faint: #3a4240;

  --accent: #4a7c59;
  --accent-dim: #2d4a38;
}
```

## Цветовая логика

### `--bg`

Основной фон.
Почти чёрный с зелёным подтоном.
Он создаёт ощущение ночного леса.

### `--surface`

Карточки и панели.
Должны быть едва заметными, как матовое стекло в темноте.

### `--border`

Тонкие границы интерфейса.
Граница должна помогать читать структуру, но не привлекать внимание.

### `--text-primary`

Главный текст.
Не использовать чистый белый. Интерфейс должен быть мягким.

### `--text-muted`

Вторичный текст, подписи, labels, inactive states.

### `--text-faint`

Самый тихий текст.
Использовать для цитат, фоновых подписей, будущих состояний.

### `--accent`

Единственный активный цвет.
Использовать редко: прогресс, выбранные состояния, текущая точка пути.

### `--accent-dim`

Приглушённый зелёный для мягких hover-состояний, пройденных линий и спокойных подсветок.

# Типографика

Основной шрифт:

```css
font-family: 'Inter', sans-serif;
```

Типографика должна быть:

* тонкой;
* спокойной;
* просторной;
* без декоративности;
* без агрессивного веса.

## Логотип

```css
font-size: 14px;
font-weight: 500;
letter-spacing: 0.4em;
text-transform: uppercase;
color: var(--text-primary);
```

Текст:

```txt
LOST SIGNAL
```

## Большое число

Используется для количества дней.

```css
font-size: 160px;
font-weight: 300;
letter-spacing: -4px;
line-height: 0.9;
color: var(--text-primary);
```

Число должно быть главным визуальным якорем экрана.

## Подписи

```css
font-size: 11px;
font-weight: 600;
letter-spacing: 0.2em;
text-transform: uppercase;
color: var(--text-muted);
```

Примеры:

```txt
ТЕКУЩАЯ ЛОКАЦИЯ
ШУМ ПОЗАДИ
НАЧАЛО
ПЕРВЫЙ РУБЕЖ
```

## Значения

```css
font-size: 22px;
font-weight: 400;
color: #d9dbd8;
```

Примеры:

```txt
Старая хижина
короткие видео
62 часа
```

## Цитаты и тихие фразы

```css
font-size: 13px;
font-style: italic;
color: var(--text-faint);
text-align: center;
```

Примеры:

```txt
Шум стал тише. Лес гуще.
```

```txt
Сигнал почти исчез. Путь продолжается.
```

# Интерфейсные принципы

## 1. Минимум элементов

Если элемент не усиливает тишину, путь или ясность — его нужно убрать.

## 2. Атмосфера важнее украшений

Фон, туман, карточки и текст должны работать вместе.
Нельзя добавлять декор ради декора.

## 3. Никакой геймификации

Не использовать:

```txt
стрик
ачивка
уровень
челлендж
награда
серия
цель выполнена
```

Lost Signal не должен ощущаться как игра или habit tracker.

## 4. Мягкая конкретика

Интерфейс должен быть атмосферным, но понятным.

Плохо:

```txt
31 день вдали
```

Лучше:

```txt
31 день без коротких видео
```

Плохо:

```txt
Цель выполнена!
```

Лучше:

```txt
Lost Signal достигнут
```

## 5. Спокойные действия

Кнопки не должны кричать.

Плохо:

```txt
СТАРТ!
```

Лучше:

```txt
Начать путь
```

Плохо:

```txt
Сбросить прогресс
```

Лучше:

```txt
Начать заново
```

# Layout-направление

Основной экран должен строиться вокруг одной центральной карточки.

Структура:

```txt
[Navbar]
  LOST SIGNAL
  тихая подпись
  навигация

[Main Card]
  большое число дней
  что пользователь оставил позади
  возвращённое время
  текущая локация
  дата начала
  первый рубеж
  journey progress

[Footer]
  тихая фраза
```

Главный экран не должен быть перегружен действиями.
Он должен быть созерцательным.

---

# Карточки

Основной стиль:

```css
background: var(--surface);
border: 1px solid var(--border);
border-radius: 16px;
backdrop-filter: blur(12px);
padding: 48px;
```

Карточки должны ощущаться как стекло поверх тёмного леса.

Не использовать:

* плотные белые блоки;
* яркие карточки;
* кислотные акценты;
* тяжёлые тени;
* лишние обводки;
* декоративные градиенты без смысла.

---

# Фон

Фон — важная часть продукта.

Он должен создавать ощущение:

```txt
лес далеко
шум позади
свет слабый
тишина рядом
```

Допустимые решения:

* тёмная фотография леса в тумане;
* SVG-силуэты елей;
* CSS-gradients;
* fog/noise layers;
* затемняющий overlay.

Рекомендуемый overlay:

```css
background:
  linear-gradient(
    to bottom,
    rgba(13, 15, 14, 0.6),
    rgba(13, 15, 14, 0.85)
  );
```

Фон не должен конкурировать с текстом.
Лес должен читаться слабо, почти как память.

# Journey

Путь состоит из шести локаций:

```txt
Forest Edge
Deep Pines
Old Cabin
Foggy Lake
Silent Valley
Lost Signal
```

Русские названия:

```txt
Опушка
Глубокие сосны
Старая хижина
Туманное озеро
Тихая долина
Lost Signal
```

Journey progress — это не обычный progress bar.
Это карта удаления от шума.

Пройденные точки показывают дистанцию.
Текущая точка показывает, где пользователь сейчас.
Будущие точки должны быть тихими и едва заметными.

# Иконки

Использовать простые outline-иконки.

Рекомендуемый стиль:

```txt
lucide-react
```

Размер:

```css
width: 18px;
height: 18px;
```

Цвет:

```css
color: #4a5450;
```

Активные состояния:

```css
color: var(--accent);
```

Иконки не должны быть яркими.
Они только помогают ориентироваться.

# Tone of voice

Lost Signal говорит коротко.

Без давления.
Без восторга.
Без мотивационных лозунгов.

## Использовать

```txt
шум
тишина
путь
сигнал
дистанция
рубеж
маршрут
локация
время возвращено
начать путь
уйти дальше
```

## Не использовать

```txt
привычка
цель
челлендж
стрик
продуктивность
мотивация
достижение
самоконтроль
```

## Примеры

Плохо:

```txt
Выберите привычку, которую хотите бросить
```

Хорошо:

```txt
Что для тебя шум?
```

Плохо:

```txt
Сколько времени вы тратили на плохую привычку?
```

Хорошо:

```txt
Сколько времени шум забирал в день?
```

Плохо:

```txt
Вы достигли цели
```

Хорошо:

```txt
Lost Signal достигнут
```

# Onboarding-направление

Первый вход должен ощущаться как начало пути, а не как настройка приложения.

Первый экран:

```txt
LOST SIGNAL

Уйти от шума.
```

Основное действие:

```txt
Начать путь
```

Onboarding должен быть:

* fullscreen;
* спокойный;
* с одним главным вопросом на экран;
* без лишних объяснений;
* с мягким progress indicator;
* без ощущения анкеты.

---

# Главная проверка дизайна

Перед любым изменением интерфейса задать вопросы:

1. Это стало тише?
2. Это стало понятнее?
3. Это похоже на Lost Signal?
4. Это не превратилось в habit tracker?
5. Это можно убрать?

Если ответ на пятый вопрос “да” — скорее всего, нужно убрать.
